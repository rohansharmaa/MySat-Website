OBC

Allsubsystems on board MySat are controlled by an On-board Computer (OBC). OBC processes telecommands received from Earth, monitors the current state of the satellite, automatically performs scheduled tasks and collects telemetry data from all subsystems and cameras. It is based on the CubeComputer V3 platform from CubeSpace , which is based on the EFM32 microcontroller (ARM Cortex-M3). The data is stored in an external Flash memory which is on the Payload plate (PLD).

Construction of on-board computer

OBC is equipped with parallel Flash, StaticRAM and EEPROM memory. The Flash memory stores software images whose integrity is checked by the bootloader, which then loads the functional image into the internal Flash memory of the EFM32 microcontroller. If it is not possible to load a valid image of the software, the on-board computer will load the special operating mode from the EEPROM memory and will try to repair the software.

The StaticRAM memory, on the other hand, stores the current OBC results. It is secured by hardware redundancy, Hamming code (EDAC) implemented by FPGA and scrubbing implemented by OBC software.

Tests

The on-board computer software is complex and has many dependencies. It is based on the FreeRTOS real-time system . The right test stack and test environment enabled convenient testing of the software at all times.Unit tests were created using GoogleTest and GoogleMock libraries (with minor modifications). In addition, some tests were created using the RapidCheck library supporting the property-based testing technique. Regular code review and continuous integration helped maintain high quality code throughout the entire software development period. A set of end-to-end tests (Python) allowed to confirm the readiness of the software and eliminate hard to detect errors.

Experiment data and historical status (up to ~ 36 hours) of the satellite are stored in PLD memories (external memories are connected to the Piggyback connector). There are three Flash memories with a size of 16MB each.
